User Flow

How Does an Agent Pay?

The x402 protocol turns HTTP payment interactions into a clean 3-step loop. Any agent that can sign a transaction can integrate in minutes.

1

Request the Endpoint

Your agent makes a normal GET request to any paid data endpoint. No API key needed.

# Agent requests BTC VWAP GET https://agentic-payments-production.up.railway.app/v1/vwap/BTCUSD
2

Receive the 402 Challenge

The server responds with HTTP 402. The PAYMENT-REQUIRED header contains Base64-encoded JSON with the exact USDC amount and destination wallet.

HTTP/1.1 402 Payment Required PAYMENT-REQUIRED: eyJuZXR3b3JrIjogInNvbGFuYS8... // Decoded header: [{ "network": "solana:mainnet", "payTo": "89xvy...gtMEu", "amount": 0.002, "currency": "USDC" }]
3

Pay & Submit Proof

The agent sends USDC to the settlement wallet on Solana (or Base L2), then re-submits the request with the transaction hash as a PAYMENT-SIGNATURE header.

# Resubmit with transaction proof GET /v1/vwap/BTCUSD PAYMENT-SIGNATURE: eyJwcm9vZiI6IjNOc2dp...
✓

Receive Verified Data

The server verifies the on-chain transaction via Solana RPC, then returns the full institutional data payload — enriched with LLM-readable metadata.

{ "status": "ok", "data": { "pair": "BTCUSD", "vwap": 75483.10, "volume": 8747.92, "market_cap": 1511062107770.38 }, "meta": { "provider": "Blocksize Capital", "endpoint": "Real-Time VWAP", "asset_class": "crypto" } }
