## Campaign Thesis

Blocksize should open the 14-day GTM sprint with API-native teams that already serve developers, agents, or crypto/onchain data workflows. The immediate ask is not "buy Blocksize." The ask is:

> Help us pressure-test pay-per-call financial data for AI agents using x402 and MCP.

This keeps the conversation low-friction while validating whether agentic payments can become the main revenue driver.

## First Sprint Goal

Book 8 to 12 design-partner conversations from 25 named accounts.

Primary conversion event: a prospect agrees to run or watch the 5-minute paid-call demo.

Secondary conversion event: a prospect identifies one data slice they would pay for, resell, or distribute via x402/MCP.

## Segment Messaging

### Agent Tooling Platforms

Accounts: Browserbase, Firecrawl, Exa, Vercel.

Angle: "Your users are building agents that can fetch, browse, search, and act. We are testing the missing financial-data purchase primitive: an agent hits a 402, pays per call, and receives institutional market data without account setup."

Best ask: "Would you watch a 5-minute demo and tell us whether this belongs as a tool/integration for your agents?"

### Onchain And Market Data APIs

Accounts: Dune, Kaiko, Nansen, Flipside, CoinMetrics, CoinGecko, CoinAPI, Covalent, Polygon.io, Intrinio, Twelve Data, Alpha Vantage.

Angle: "APIs are built for humans and dashboards, but agents want one-off, usage-priced data slices. We are testing whether x402/MCP can convert machine traffic into paid calls without subscriptions or API keys."

Best ask: "Which one premium query would you expose if an agent could pay for it instantly?"

### Protocol And Payment Partners

Accounts: Coinbase CDP/x402, Circle, Skyfire.

Angle: "Blocksize can be a concrete financial-data seller use case for agentic payments: real data, clear per-call pricing, demoable 402 flow, and an institutional trust roadmap."

Best ask: "What would make this a credible showcase or ecosystem integration?"

### Compliance And Payments Ops

Accounts: Modern Treasury, Plaid, Chainalysis, TRM Labs.

Angle: "Autonomous finance only works when data calls and payments have policy, proof, and auditability. We want design feedback before overbuilding."

Best ask: "What control would you need before letting an agent pay for financial or risk data?"

## Qualification Questions

- Where are agents already touching your product or API?
- Do those agents arrive as real customers, bots, or unmonetized traffic?
- Which data call would be worth $0.25, $1.00, or $5.00 if it were available on demand?
- Do you prefer direct per-call payments, prepaid credits, or usage-based monthly commitments?
- What proof would you need to trust that a paid agent request is legitimate?
- Would x402 help, or do you need AP2, cards, ACH, account-based billing, or invoices?
- What would make Blocksize useful as a partner rather than just another API?
